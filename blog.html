FAQ


1. What is the DOM?

The Document Object Model (DOM) is a programming interface that allows scripts and programs to dynamically access and update the content, structure, and style of a document, usually HTML or XML. Here's a breakdown of its key aspects: Structure Representation: The DOM represents the document as a tree structure where each node corresponds to a part of the document (like elements, attributes, and text). Interactivity: It allows developers to interact with the document using programming languages like JavaScript. This means you can add, delete, or modify elements and their properties dynamically. Event Handling: The DOM supports events (like clicks, keyboard input, etc.), allowing developers to define how the document responds to user actions. Browser Integration: Modern web browsers implement the DOM, enabling developers to manipulate web pages interactively and responsively.


2. How is the DOM different from HTML?

HTML is a markup language used to define the static structure and content of a web page, while the DOM is a dynamic representation of that structure as a tree of objects that can be manipulated by scripts like JavaScript. Essentially, HTML provides the initial content, and the DOM allows for real-time modifications.


3. What are DOM elements and nodes?

DOM elements represent HTML tags (like div, p, a), while DOM nodes are the general term for all items in the DOM tree, which includes elements, text, attributes, and comments. Every element is a node, but not every node is an element.


4.What is event delegation in the DOM?

Event delegation is a technique in JavaScript where you add a single event listener to a parent element instead of adding listeners to individual child elements. The event listener can capture events from child elements due to event bubbling. This improves performance when dealing with large numbers of elements.
